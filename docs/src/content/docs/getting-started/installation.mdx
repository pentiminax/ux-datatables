# Installation

<Aside type="note">
  Before you start, make sure [Symfony UX
  StimulusBundle](https://symfony.com/bundles/StimulusBundle/current/index.html) is configured in
  your application.
</Aside>

## Install the Bundle

<Steps>
**Install via Composer**

    ```bash
    composer require pentiminax/ux-datatables
    ```

    Symfony Flex will automatically enable the bundle.

**Install Assets**

    <Tabs>
      <TabItem label="AssetMapper">
        If you're using **AssetMapper** (recommended for Symfony 6.4+), no extra work is needed. The bundle ships DataTables assets automatically.
      </TabItem>
      <TabItem label="Webpack Encore">
        If you're using **Webpack Encore**, install the JavaScript dependencies and rebuild your assets:

        ```bash
        npm install --force
        npm run watch
        ```
      </TabItem>
    </Tabs>

**Verify Installation**

    The `@pentiminax/ux-datatables/datatable` Stimulus controller is now available in your application.

</Steps>

## Bundle Configuration

Create a configuration file at `config/packages/data_tables.yaml`:

```yaml
data_tables:
  options:
    language: en-GB
    layout:
      topStart: pageLength
      topEnd: search
      bottomStart: info
      bottomEnd: paging
    lengthMenu: [10, 25, 50]
    pageLength: 10
  template_parameters:
    class: 'table'
  extensions:
    buttons: [csv, excel, pdf, print]
    select:
      style: single
```

## Styling

The bundle ships with the DataTables default theme enabled. Check your `assets/controllers.json`:

```json
{
  "controllers": {
    "@pentiminax/ux-datatables": {
      "datatable": {
        "enabled": true,
        "autoimport": {
          "datatables.net-dt/css/dataTables.dataTables.min.css": true,
          "datatables.net-bs5/css/dataTables.bootstrap5.min.css": false
        }
      }
    }
  }
}
```

The bundle supports 7 CSS frameworks (Bootstrap 5, Bootstrap 4, Bulma, Foundation, and more). The Stimulus controller automatically detects which stylesheet is loaded and applies the matching integration.

See [Styling](../styling/) for the full setup guide.
